Introduction to Filtering.

IPC filtering is the process of intercepting IPC commands transmitted between a client and a server or between two clients. This is intended for command logging for debugging, and for command modification and injection.

IPC filtering is achieved by adding an additional layer of IPC where one already exists. The filter program intercepts incoming IPC and performs an action, normally passing the IPC on to its original destination when it's done.

Standard Filters.

All standard Resourcerver IPC filters will follow the general guidelines in this section.

Local System Filtering.

Local system filtering is done between a client and its server. This requires the filter be started in conjunction with the client itself. The filter cannot be added to an existing client/server connection.

Filtering with a standard filter requires that the filter program itself appear as the client's program name when the server starts the client, and the filter subsequently starts the client program itself after making changes to the initial IPC configuration to insert itself in the communication chain. Normally this is transparent to the client program.

Standard filter programs will execute all remaining arguments (after those it uses) as a child process. Because of this, standard filters can be stacked arbitrarily, with the arguments after the last filter becoming the actual client process. Each filter slows down IPC slightly, however, so as few filters as possible should be used. Filters should not be a standard part of a Resourcerver system: they should only be used for debugging.

Standard filter programs don't normally need to perform additional copy operations for IPC that isn't processed.

Inter-system Filtering.

Inter-system filtering takes place between two remotely-connected clients. This is done by specifying a standard filter at the time of connection between a standard socket forwarder and another of the same type.

Although the command used to start a standard filter itself doesn't require anything other than the arguments needed to set up that filter, you can also stack standard filters when filtering this type of IPC. The standard filters will execute all arguments following the arguments the filter itself uses, but those additional arguments must also execute a standard filter, otherwise the IPC connection will fail.

Unlike local system filtering, all inter-system filters must copy perform a copy on all IPC, even if that filter doesn't use it. This can slow down IPC more than inter-system filters do.
