Overview of the Guide.

This is a basic introduction and reference for both administrators and developers of Resourcerver systems. This guide covers all introductory information, most tasks required to administer a system, and concepts required to understand how the system works.

Most users reading this guide will go on to develop Resourcerver applications. The Application Development Guide is the next guide in this series and introduces developers to client program development and basic inter-client interaction. The guide you are currently reading does not explicitly cover programming-level development, but does cover shell-level concepts.

The majority of the information in this guide is conceptual, with the latter sections being dedicated to specific programs. The most complex part of the project at the user level is understanding how the system works internally. When a section of this guide outlines what a client, server, or system can and can't do, normally that means the application developer must design a system using those specifications. The generally administrator has limited control over those aspects, but should still be aware of them to understand the system better.
